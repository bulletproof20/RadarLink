# Learning Outcomes

This page records technical and personal growth during the RadarLink project — as a student working through **Estruturas Avançadas de Dados (EAD)**, not as a product claim.

## Language and memory

- **C programming** — first substantial programs written without garbage collection; learning to think in terms of pointers, stack vs heap, and explicit lifetimes.
- **Manual memory management** — allocating and freeing structures for vertices, edges, and auxiliary lists; understanding leaks and ownership at a practical level.

## Data structures and algorithms

- **Dynamic structures** — linked lists and graph adjacency representations built at runtime from file input.
- **Graphs** — modelling antennas as vertices and frequency-layer connections as edges; reasoning about subgraphs per resonance frequency.
- **DFS and BFS** — implementing depth-first and breadth-first traversals from a chosen antenna; comparing traversal order and use cases.
- **Path enumeration** — finding all paths between two antennas within the graph model.

## Software organisation

- **File processing** — loading and validating city matrices from text files; handling edge-case filenames and malformed input.
- **Modular design** — separating file I/O, general graph logic, specialised traversals, terminal interface, and (in the deliverable) extended analysis into distinct modules.
- **Technical documentation** — later experience generating **Doxygen** API reference and maintaining narrative docs with **MkDocs**, as part of repository preservation.

## Honest scope

These outcomes describe skills practiced **during the original academic work** and the **documentation phase** that followed. They do not imply that the checkpoint or deliverable source was rewritten or modernised after evaluation — see [Repository Evolution](../preservation/repository-evolution.md).
